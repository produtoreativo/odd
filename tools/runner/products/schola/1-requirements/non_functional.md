### Segurança
O Schola deve garantir autenticação e autorização dos usuários, utilizando o Certificare para autenticar os usuários. Além disso, o sistema deve garantir a integridade das informações, evitando duplicação de cadastro local.

### Privacidade
O Schola deve respeitar a LGPD, armazenando dados pessoais dos alunos e administradores com segurança e sigilo.

### Confiabilidade
O sistema deve ter disponibilidade alta, garantindo que os usuários possam acessar as turmas abertas para matrícula. Além disso, o Schola deve ser tolerante a falhas, permitindo que os usuários continuem a usar o sistema mesmo em caso de problemas técnicos.

### Performance
O Schola deve ter performance alta, garantindo que os usuários possam acessar as turmas abertas para matrícula rapidamente. Além disso, o sistema deve ser capaz de lidar com um grande número de solicitações simultâneas.

### Auditoria
O Schola deve registrar todas as ações dos usuários, incluindo solicitação de matrícula e confirmação de pagamento. Além disso, o sistema deve permitir que os administradores visualize as informações registradas.

### Observabilidade
O Schola deve ter logs e métricas para monitorar o desempenho do sistema. Além disso, o sistema deve enviar eventos para um servidor de mensagens, permitindo que os administradores sejam notificados em caso de problemas técnicos.
